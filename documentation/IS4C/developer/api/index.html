POS Module API


as of: April 10, 2013

POS is designed to be altered and extended via the use of modules. The structure is object oriented so modules are classes and they are discovered via the parent class(es) they inherit from. This document describes the modules that are available.

  • Prerequisites:
    • The AutoLoader
  • Available Modules:
    • Kicker
    • ProductSearch
    • SpecialDept
    • TenderReport
    • Receipt Building
      1. DefaultReceiptDataFetch
      2. DefaultReceiptFilter
      3. DefaultReceiptSort
      4. DefaultReceiptTag
      5. DefaultReceiptFormat
