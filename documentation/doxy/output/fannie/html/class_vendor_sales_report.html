VendorSalesReport::fetch_report_data ( )

Report Query notes: Combining vendorName and prodExtra.distributor is a nod to legacy data. Eventually data should be fully normalized on products.default_vendor_id Excluding prodExtra.distributor empty string combines those records with SQL NULL. Having two different "blank" rows is confusing for users. Joins are only needed is a super department condition is involved. WHERE clause changes similarly.
